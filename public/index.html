⚡ Multi-dimensional Risk Engine
The inspection middleware processes each incoming request and aggregates risk points dynamically:
  • Velocity Thresholds: Hit counter in 5-minute sliding window. Hits > 10 adds 30 pts. Hits > 20 adds another 40 pts.
  • cURL CLI Agent check: Requests containing curl inside User-Agent header add 40 pts.
  • Missing Accept header: Browser-less automated API scrapers usually omit this header, adding 20 pts.
  • Sensitive Path Matching: Scopes matching keywords like admin add 30 pts.
if (totalRiskScore >= 70) => Quarantines fingerprint
🎯 Distributed Active Caching & Ban Policy
System locks abusive endpoints inside Redis instantly without database lookups:
  • Fingerprint (FP) Quarantine: Prevents access for 10 minutes (TTL: 600s). Key design: ban:fp:{sha256}
  • IP Abuse Tracking: Increments IP threat score ip:risk:{ip} for every fingerprint block.
  • Network Escalation Block: If an IP address registers 3 banned fingerprints within 15 minutes, the system escalates to an IP-wide ban for 30 minutes. Key: ban:ip:{ip}
📊 DynamoDB Logging Format
All telemetry is preserved asynchronously in `api-request-logs` table with zero impact on request latency: Item: { ipAddress: { S: clientIp }, timestamp: { S: new Date().toISOString() }, fingerprint: { S: fingerprint }, hits: { N: hits.toString() }, riskScore: { N: riskScore.toString() }, path: { S: req.originalUrl }, userAgent: { S: userAgent || "unknown" } }
